# rootfs 目录与 Git 策略

> **打包**: `script/rootfs/README.txt` · **方案**: [`INITRAMFS_PLAN.md`](INITRAMFS_PLAN.md) §8

---

## 目录是什么

`rootfs/` 是 **initramfs（cpio）的源树**，由 `make rootfs` → `script/rootfs/build_cpio.sh` 打成 `build/rootfs.cpio`，再链进内核镜像。

两层内容（不要混为一谈）：

| 类型 | 路径示例 | 来源 | Git |
|------|----------|------|-----|
| **测例数据（fixtures）** | `text.txt`、`cpio_probe.txt`、`mnt/` | 仓库维护、手改 | **应提交** |
| **测例程序（generated）** | `tests/*` ELF、`tests/manifest` | `make user` 生成 | **不提交** |
| **可选 busybox** | `bin/busybox` | `build_busybox.sh` 本地构建 | **不提交**（可选） |

manifest 是 CPIO 上的普通文件 `/tests/manifest`，与 ELF 一样由 `vfs_kern_read_file_slice` 加载（见 [`FILE_LOADING.md`](FILE_LOADING.md)）。

---

## .gitignore 建议

**不要把整个 `rootfs/` 写进 gitignore。**

fixtures 需要进仓库，否则干净 clone 后 `make rootfs` 打出来的 cpio 缺 `text.txt` 等，open/read 测例会失败。

当前 `.gitignore` 规则（推荐保持）：

```gitignore
# Generated by make user (cpio test ELFs + manifest)
rootfs/tests/*
!rootfs/tests/.gitkeep

# Optional local busybox
rootfs/bin/*
!rootfs/bin/.gitkeep
```

含义：

- `rootfs/tests/*`：所有 `make user` 产物忽略；
- `!rootfs/tests/.gitkeep`：保留空目录占位；
- fixtures（`text.txt`、`mnt/`、`cpio_probe.txt`）**不在 ignore 里**，应 `git add` 进仓库。

---

## 常用命令

```bash
make user ARCH=x86_64      # 构建测例 ELF → rootfs/tests/ + manifest
make rootfs ARCH=x86_64    # 刷新 build/rootfs.cpio
make ARCH=x86_64 build run
```

若 `git status` 显示 `rootfs/` untracked：通常是 fixtures 尚未 add；`tests/` 里大量 ELF 应被 ignore，只剩 `.gitkeep` 可见。
